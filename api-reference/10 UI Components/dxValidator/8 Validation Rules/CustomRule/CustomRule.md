##### shortDescription
A rule with custom validation logic.

---
To specify the custom rule, set the [type](/api-reference/10%20UI%20Widgets/dxValidator/8%20Validation%20Rules/CustomRule/type.md '/Documentation/ApiReference/UI_Widgets/dxValidator/Validation_Rules/CustomRule/#type') to *"custom"* and declare the [validationCallback](/api-reference/10%20UI%20Widgets/dxValidator/8%20Validation%20Rules/CustomRule/validationCallback.md '/Documentation/ApiReference/UI_Widgets/dxValidator/Validation_Rules/CustomRule/#validationCallback') function.

You can also set a custom [message](/api-reference/10%20UI%20Widgets/dxValidator/8%20Validation%20Rules/CustomRule/message.md '/Documentation/ApiReference/UI_Widgets/dxValidator/Validation_Rules/CustomRule/#message'), specify whether [empty values are valid](/api-reference/10%20UI%20Widgets/dxValidator/8%20Validation%20Rules/CustomRule/ignoreEmptyValue.md '/Documentation/ApiReference/UI_Widgets/dxValidator/Validation_Rules/CustomRule/#ignoreEmptyValue'), and whether the rule should be [re-evaluated](/api-reference/10%20UI%20Widgets/dxValidator/8%20Validation%20Rules/CustomRule/reevaluate.md '/Documentation/ApiReference/UI_Widgets/dxValidator/Validation_Rules/CustomRule/#reevaluate'), even if the target value is the same.
