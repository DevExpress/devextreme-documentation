DevExtreme libraries (except [DevExtreme Map](/Documentation/Guide/Common/Security_Considerations/#Content_Security_Policy/CSP_Directives_for_Map_Integrations)) use CSSOM APIs to apply and update styles at runtime. These APIs include **element**.**style**.[cssProperty](https://developer.mozilla.org/en-US/docs/Web/CSS/Reference) and **element**.**style**.[cssText](https://developer.mozilla.org/en-US/docs/Web/API/CSSStyleDeclaration/cssText) assignments.

For example, DevExtreme's internal DOM adapter uses a custom [setAttribute](https://github.com/DevExpress/DevExtreme/blob/64129d8d82144b9aa40f0c03a223e40f41699a97/packages/devextreme/js/__internal/core/m_dom_adapter.ts#L140-L146) function that updates element styles through the CSSOM **cssText** property. DevExtreme UI components also use CSSOM APIs (such as [adoptedStyleSheets](https://developer.mozilla.org/en-US/docs/Web/API/ShadowRoot/adoptedStyleSheets) and [insertRule](https://developer.mozilla.org/en-US/docs/Web/API/CSSStyleSheet/insertRule)) to apply component styles inside [Shadow DOM](/Documentation/Guide/Common/Shadow_DOM/) environments.

DevExtreme libraries do not inject `<style>` elements at runtime. DevExtreme-powered web application can enforce a Content Security Policy with the `style-src 'self'` directive without enabling the less secure `'unsafe-inline'` style source.

DevExtreme CI workflows validate [JS/TS technical demos](https://js.devexpress.com/Demos/WidgetsGallery/) (jQuery, Angular, Vue, and React) against the `style-src 'self'` directive without `'unsafe-inline'`. However, this validation does not guarantee that future releases will always remain compatible with this policy. Configure a report-only [securitypolicyviolation](https://developer.mozilla.org/en-US/docs/Web/API/Document/securitypolicyviolation_event) listener in your CI pipeline to detect CSP violations and verify that your application continues to comply with `style-src 'self'` without `'unsafe-inline'`.
